# WooCommerce Menu Hub

Универсальный консолидатор админ-меню WordPress/WooCommerce. Любой пункт меню любого плагина можно **скрыть** или **встроить как вкладку** в единый хаб — всё через UI, без правки кода чужих плагинов.

## Зачем

Когда установлено много плагинов (доставка, синхронизация, аналитика…), каждый вешает свой пункт в меню — сайдбар превращается в кашу. Этот плагин даёт навести порядок: оставить один хаб с вкладками, остальное скрыть.

## Как пользоваться

1. Активируйте плагин.
2. **Настройки → WC Menu Hub**.
3. Для каждого пункта меню выберите действие:
   - **Показать** — оставить как есть (по умолчанию);
   - **Скрыть** — убрать из меню (страница остаётся доступной по прямой ссылке);
   - **Встроить** — убрать из меню и показать как вкладку в едином хабе.
4. Задайте название и иконку хаба, сохраните.

Встроенные пункты собираются в один top-level пункт меню (по умолчанию «📦 Отправка») с горизонтальными вкладками.

## Как устроено «встраивание» (важно)

Страница встроенного плагина **не переносится** в другое место меню. Перенос сменил бы её hook-suffix и сломал бы её screen-specific JS (кнопки «тест связи», загрузка сущностей и т.п.).

Вместо этого:
1. Пункт убирается из меню через `remove_submenu_page` / `remove_menu_page`. При этом сама страница **остаётся доступной по URL** (`admin.php?page=…`), и её скрипты грузятся как раньше (hook не меняется).
2. Регистрируется один хаб-пункт.
3. Поверх всех встроенных страниц через `in_admin_header` рисуется общая панель вкладок.

Итог: единое меню с вкладками, при этом функциональность каждой страницы полностью сохраняется.

## Безопасность

- Пункты `options-general.php` (Настройки), `index.php` (Консоль), сама страница плагина и хаб — **защищены** и не могут быть скрыты (защита от самоблокировки).
- Скрытые страницы не удаляются — доступны по прямой ссылке и возвращаются в меню, как только снимете «Скрыть».
- Конфигурация хранится в одной опции `wphub_config`; деактивация плагина возвращает все пункты на место.

## Требования

- WordPress 5.x+, PHP 7.4+.
- Для пунктов с `manage_woocommerce` нужен WooCommerce (хаб использует эту capability).
